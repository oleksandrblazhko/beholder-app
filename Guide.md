## Опис вузлів

### Вузол Detection->Webcam Detection
вузол, що забезпечує джерело даних з веб-камери для виявлення маркерів:
1) вихідний параметр:
- FEED - джерело даних (feed), яке містить відеопотік з веб-камери та результати виявлення маркерів
Вузол є основним джерелом вхідних даних для системи виявлення маркерів. Він забезпечує відеопотік, який використовується іншими вузлами для виявлення та відстеження ArUco-маркерів.

### Вузол Marker->DetectMarker
розпізнає ArUco-маркер у полі зору камери:
1) вхідні параметри:
- ID - номер ArUco-маркера, який визначається через додатковий блок variables->Number;
- Source - джерело даних, яке визначається через підключення блока Detection->WebCameraDetection;
- Timeout - час у мілісекундах, після якого маркер вважається відсутнім, якщо він не був виявлений знову впродовж цього часового вікна;
2) вихідні параметри:
- Detect - true/false
- X, Y - горизонтальна та вертикальна координати центру маркера, початок координат (0,0) знаходиться в верхньому лівому куті зображення;
- Angle - кут повороту маркера, за умови, що кути нумеруються за годинниковою стрілкою (0, 1, 2, 3), починаючи з верхнього лівого кута,
у горизонтальному положенні маркера кут дорівнює 0, а його повертання направо збільшує значення кута.
Блок є ключовим для побудови інтерактивних систем, що керуються фізичними маркерами,
дозволяючи відстежувати розташування, орієнтацію та наявність конкретного маркера в полі зору камери, що може
бути використано для керування інтерфейсами, ігровими об'єктами або іншими елементами програми.

### Вузол Variables->Number
вузол, що виводить числове значення, яке можна встановити вручну за допомогою вбудованого числового поля:
1) вихідний параметр:
- value - числове значення (number), яке встановлюється через вбудоване числове поле вузла
Вузол використовується для задання постійних числових значень, які можуть бути використані іншими вузлами у системі, наприклад, як ID маркера, порогові значення або інші параметри.

### Вузол Logic->Greater Then Gate
виконує операцію порівняння "більше ніж" (>) між двома числовими значеннями:
1) вхідні параметри
- A - перше числове значення для порівняння, чи воно більше за значення B.
- B - друге числове значення, з яким порівнюється значення A (порогове значення).
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо A > B, і "false", якщо A ≤ B

### Вузол Logic->Less Than Gate
виконує операцію порівняння "менше ніж" (<) між двома числовими значеннями:
1) вхідні параметри:
- A - перше числове значення для порівняння, чи воно менше за значення B;
- B - друге числове значення, з яким порівнюється значення A (порогове значення);
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо A < B, і "false", якщо A ≥ B
Вузол використовується для прийняття рішень на основі порівняння двох значень, аналогічно до Greater Than Gate, але з протилежною операцією порівняння.

### Вузол Logic->Between Gate
виконує операцію перевірки, чи знаходиться значення в заданому діапазоні (A < x < B):
1) вхідні параметри:
- A - нижня межа діапазону;
- X - значення, яке перевіряється на належність до діапазону;
- B - верхня межа діапазону;
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо A < X < B, і "false" в інших випадках
Вузол використовується для перевірки, чи знаходиться певне значення (наприклад, координата маркера) в заданому діапазоні між двома пороговими значеннями.

### Вузол Logic->AND Gate
виконує логічну операцію "І" (AND) між двома булевими значеннями:
1) вхідні параметри:
- A - перше булеве значення для логічної операції;
- B - друге булеве значення для логічної операції;
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо A = true І B = true, і "false" в усіх інших випадках
Вузол використовується для умовного логічного управління, коли необхідно, щоб дві умови були одночасно істинними для активації наступних дій.

### Вузол Logic->NOT Gate
виконує логічну операцію заперечення "НЕ" (NOT) над одним булевим значенням:
1) вхідний параметр:
- value - булеве значення для логічної операції;
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо value = false, і "false", якщо value = true
Вузол використовується для інвертування логічних значень у системі, тобто змінює значення на протилежне.

### Вузол Logic->OR Gate
виконує логічну операцію "АБО" (OR) між двома булевими значеннями:
1) вхідні параметри:
- A - перше булеве значення для логічної операції;
- B - друге булеве значення для логічної операції;
2) вихідний параметр:
- Trigger - логічне значення (bool), яке дорівнює "true", якщо A = true АБО B = true (або обидва), і "false" лише тоді, коли обидва значення хибні
Вузол використовується для умовного логічного управління, коли достатньо, щоб одна з двох умов була істинною для активації наступних дій.

### Вузол Logic->Value Change Trigger
відстежує накопичену зміну вхідного значення з часом (зберігається в totalDelta),
порівнює поточне вхідне значення з попередньо записаним значенням, щоб обчислити різницю,
ця різниця додається до поточного підсумку (totalDelta):
1) вхідні параметри
- Value - вхідне значення, для якого треба відстежувати його зміни;
- Threshold - порогове значення змін, після перевищення якого активується тригер
2) вихідний параметр:
- Trigger - стан активатиції тригера, коли поріг перевищено, тоді значення "true", інакше - "false"

### Вузол Logic->Angle Change Trigger
відстежує зміни кутового значення (наприклад, кута повороту маркера) і активації виходу, 
коли накопичена зміна кута перевищує заданий поріг,
а для обчислення різниці між поточним і попереднім кутом використовується векторна математика, 
що дозволяє правильно обробляти обертання кута, наприклад, коли кут змінюється від 359 градусів до 1 градуса,  
це буде враховано як зміна на 2 градуси, а не на -358 градусів):
1) вхідні параметри:
- Angle - вхідне кутове значення, для якого треба відстежувати його зміни;
- Threshold - порогове значення змін, після перевищення якого активується тригер
2) вихідний параметр:
- Trigger - стан активатиції тригера, коли поріг перевищено, тоді значення "true", інакше - "false"

### Вузол Logic->Periodic
генерує періодичні сигнали (імпульси), перемикаючись між станами "true" і "false" з заданим інтервалом часу,
коли вузол активний (ACTIVE = true) і накопичений час (totalDt) досягає половини від PERIOD, 
вузол перемикає свій внутрішній стан (isOn) на протилежний,
а після перемикання лічильник часу скидається, 
наприклад, якщо встановити PERIOD = 1000 (1 секунда) і ACTIVE = true, вузол буде видавати значення "true" протягом 0.5 секунди, 
потім "false" протягом 0.5 секунди, і так далі з періодом 1 секунда:
1) вхідні параметри:
- Active - логічний параметр, який вмикає або вимикає роботу вузла. Якщо ACTIVE = true, вузол активний і генерує періодичні сигнали. Якщо ACTIVE = false, вузол не генерує сигнали.
- Period - числовий параметр, який визначає тривалість одного повного циклу (в мілісекундах). Це час, необхідний для переходу вузла з одного стану в інший і назад (наприклад, з "false" в "true" і назад в  
        "false").  
2) вихідний параметр:
- Trigger - булевий (bool) вихід, який періодично перемикається між "true" і "false" з заданим періодом, коли вузол активний.
Вузол корисний для створення автоматичних повторюваних подій, таких як:
- періодичне натискання клавіш;
- створення миготливих ефектів;
- автоматичне повторення дій з певною частотою;
- синхронізація дій з часовими інтервалами.

### Вузол Marker->Marker Pair
виконує обчислення параметрів між двома маркерами:
1) вхідні параметри:
- ID_A - ID першого маркера, який буде використовуватися як опорний;
- ID_B - ID другого маркера, позиція якого визначається відносно першого;
- Source - джерело даних, яке визначається через підключення блока Detection->WebCameraDetection;
2) вихідні параметри:
- Distance - відстань між двома маркерами;
- AngleBetween - кут між двома маркерами;
- Rel_Pos_X - відносна позиція маркера B відносно маркера A по осі X;
- Rel_Pos_Y - відносна позиція маркера B відносно маркера A по осі Y;
Вузол використовується для аналізу відносного розташування двох маркерів, що дозволяє створювати більш складні інтерактивні системи, які залежать від взаємного розташування фізичних об'єктів.

### Вузол Keys->Press Key
утримує клавішу натиснутою протягом усього часу, поки надходить істинне значення 
(подія DOWN при активації та подія UP при деактивації);

### Вузол Keys->Tap Key
виконує одноразове натискання клавіши, коли отримує істинне значення
(DOWN та UP події відбуваються одразу при отриманні істинного значення)
